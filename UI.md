## 🎯 核心方法论

### 提示词工程的关键要素

1. **明确的设计目标**
   - 定义项目类型和受众
   - 明确设计风格参考（如：AWS 官网、Linear.app）

2. **详细的设计规范**
   - 视觉规范：配色、字体、间距、圆角
   - 技术规范：框架选择、性能要求

3. **交互式引导流程**
   - 采用多轮对话，而非一次性输入
   - 逐步细化需求，让 AI 主动提问

4. **具体的技术实现细节**
   - 明确保留的元素和数据
   - 明确技术栈和限制条件

使用以下提示词模板与 Claude Code 进行多轮对话：

```
我有一个 AWS GPU 实例对比网站（/home/ubuntu/codes/aws-gpu/index.html），
当前是静态表格展示。请作为产品设计大师，帮我重新设计：

**设计目标：**
- 专业的云服务产品展示页面
- 参考 AWS 官网风格 + 现代化数据表格设计

**必须保留的元素：**

**改进方向（请逐步询问我）：**

**技术要求：**
- 纯 HTML + CSS（或建议升级到 Tailwind？）
- 不依赖 React/Vue 等框架（除非我同意）
- 保持轻量级，加载速度快

请先问我：您最想改进的是哪个方面？
```

## 🎨 设计风格参考

### 推荐配色方案

**AWS 官方风格：**
- 主色：`#232F3E`（深蓝灰）
- 强调色：`#FF9900`（橙色）
- 背景：`#F5F5F5`（浅灰）
- 文字：`#16191F`（深灰黑）

**现代化方案（Glassmorphism）：**
- 主色：`#1E293B`（石板蓝）
- 次要色：`#F8FAFC`（浅白）
- 玻璃态效果：`rgba(255, 255, 255, 0.1)` + `backdrop-filter: blur(10px)`
- 渐变背景：`linear-gradient(135deg, #667eea 0%, #764ba2 100%)`

### 布局参考

- **AWS 官网：** 简洁、专业、大量留白
- **Linear.app：** 现代、扁平、微交互动画
- **Notion：** 清晰的层级结构、舒适的阅读体验

## 📝 实施步骤

### Step 1: 评估现状
```bash
# 查看当前文件结构
ls -lh /home/ubuntu/codes/aws-gpu/
```

当前状态：
- ✅ 静态 HTML + CSS
- ✅ 表格式数据展示
- ✅ 基础响应式设计
- ❌ 无交互功能
- ❌ 无数据筛选
- ❌ 无可视化图表

### Step 2: 选择技术方案

**选项 1：保持纯静态（推荐）**
- HTML + CSS + Vanilla JavaScript
- 优点：轻量、快速、无依赖
- 适合：当前项目规模

**选项 2：引入 Tailwind CSS**
- 需要 npm + build 流程
- 优点：快速开发、一致的设计系统
- 适合：未来扩展需求

**选项 3：升级到 React/Next.js**
- 需要重构整个项目
- 优点：组件化、动态数据加载
- 适合：需要复杂交互和后端集成

### Step 3: 渐进式改进

**第一阶段：视觉优化（1-2 小时）**
1. 更新配色方案
2. 优化字体和间距
3. 添加 hover 效果
4. 响应式优化

**第二阶段：功能增强（2-4 小时）**
1. 添加搜索框（JavaScript 实现）
2. 添加筛选器（下拉菜单）
3. 添加排序功能（点击表头）

**第三阶段：高级功能（4-8 小时）**
1. 实例对比功能
2. 数据可视化图表（Chart.js）
3. 深色模式切换
4. 动画和微交互

## 💡 提示词模板库

### 模板 1：视觉优化

```
优化 /home/ubuntu/codes/aws-gpu/index.html 的视觉设计：

1. 配色方案：使用 AWS 官方配色（#232F3E + #FF9900）
2. 表格优化：增加行高、优化对齐、添加斑马纹
3. 添加效果：hover 高亮、平滑过渡动画

要求：
- 只修改 CSS，不改变 HTML 结构
- 保持所有现有功能
- 代码注释清晰
```

### 模板 2：功能增强

```
为 /home/ubuntu/codes/aws-gpu/index.html 添加交互功能：

功能需求：
1. 搜索框：实时过滤实例（按名称或 GPU 型号）
2. 筛选器：
   - 工作负载（训练/推理）
   - 区域（Global/China）

技术要求：
- 使用 Vanilla JavaScript（不引入框架）
- 不破坏现有样式
- 添加清空筛选按钮
- 显示筛选结果数量
```

### 模板 3：数据可视化

```
为 /home/ubuntu/codes/aws-gpu/index.html 添加数据可视化：

需求：
1. 顶部统计卡片：总实例数、最新发布、价格区间
2. 价格对比柱状图（Chart.js）：横向对比各系列平均价格
```
