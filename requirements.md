# AWS GPU 实例展示项目 - 需求文档

## 项目概述

本项目旨在创建一个静态网页，全面展示 AWS（Amazon Web Services）目前提供的所有 GPU 实例类型，包括详细的技术规格、性能特点和适用场景，帮助用户快速了解和选择适合自己需求的 GPU 实例。

## 项目目标

- 为销售人员提供一个直观、易用的 AWS GPU 实例参考指南
- 帮助用户根据具体应用场景快速筛选和对比合适的 GPU 实例
- 提供实时更新的 GPU 价格和可用区域信息
- 降低用户在选择 AWS GPU 实例时的决策成本

## 功能需求

### 1. GPU 实例信息展示

#### 1.1 实例类型列表
展示以下 AWS GPU 实例系列：
- **P 系列** - 通用 GPU 计算实例（用于训练和推理）
- **G 系列** - 图形密集型实例（用于渲染和轻量推理）
- **Inf 系列** - 机器学习推理专用实例（AWS Inferentia）
- **Trn 系列** - 机器学习训练专用实例（AWS Trainium）
- **DL 系列** - 深度学习实例（Habana Gaudi）

> **注意**：具体实例型号和规格详见 [gpu.md](./gpu.md)

#### 1.2 每个实例的详细信息
对于每个实例类型，需要展示：
- **实例名称**：如 p5.48xlarge, g5.xlarge 等
- **GPU 型号**：如 NVIDIA H100, A100, V100, T4, AMD Radeon Pro 等
- **GPU 数量**：1个、2个、4个、8个等
- **GPU 显存**：每个 GPU 的显存大小（GB）
- **总显存**：所有 GPU 的总显存
- **可用区域**：支持的 AWS 区域

### 2. 适用场景分类

#### 2.1 场景分类
按使用场景对实例进行分类和推荐：

**机器学习训练**
- 大规模深度学习模型训练
- 分布式训练
- 按模型规模和预算提供多层级推荐
- 包含顶级性能、高性能、通用、经济型等不同档位
- 支持AWS自研芯片作为性价比选项

> **详细推荐**：参见 [gpu.md](./gpu.md) 中的"AI训练场景"章节

**机器学习推理**
按模型规模细分推荐：

- **超大模型推理**（70B-480B+ 参数）
  - 适用于 GPT-3、Claude 等级别模型
  - 需要高显存和高带宽支持

- **中大型模型推理**（20B-70B 参数）
  - 平衡性能与成本
  - 主流企业级应用场景

- **小型模型推理**（<20B 参数）
  - 区分实时推理和批量推理
  - 提供GPU和AWS自研芯片两种选择

- **高吞吐量推理**
  - 极致性能需求场景
  - 金融交易、实时对话等

> **详细推荐**：参见 [gpu.md](./gpu.md) 中的"AI推理场景"章节

**图形渲染与可视化**
- 3D 渲染
- 视频编码/转码
- 虚拟工作站
- CAD/CAE 应用

**高性能计算（HPC）**
- 科学计算
- 金融建模
- 基因组学
- 天气预报

**AI 研究与开发**
- 研究型工作负载
- 模型实验
- 快速原型开发

**游戏开发与云游戏**
- 游戏流式传输
- 游戏开发测试
- 实时渲染

> **详细推荐**：所有场景的具体实例推荐参见 [gpu.md](./gpu.md)

### 3. 筛选与对比功能

#### 3.1 筛选器
用户可以通过以下维度筛选实例：
- GPU 型号（按具体GPU芯片型号筛选）
- 实例系列（P, G, Inf, Trn, DL）
- 使用场景（训练、推理、渲染等）
- GPU 数量（单卡、多卡配置）
- 价格范围（按小时成本区间）
- 区域可用性（按AWS区域筛选）

#### 3.2 实例对比
- 支持选择 2-4 个实例进行并排对比
- 高亮显示差异项
- 显示性能/价格比

#### 3.3 排序功能
- 按价格排序（低到高、高到低）
- 按 GPU 性能排序
- 按显存大小排序
- 按发布时间排序（最新、最旧）

### 4. 用户体验功能

#### 4.1 搜索功能
- 支持按实例名称快速搜索
- 支持按 GPU 型号搜索
- 模糊搜索支持

#### 4.2 收藏与分享
- 用户可以收藏常用的实例类型（本地存储）
- 生成分享链接（URL 参数）

#### 4.3 响应式设计
- 适配桌面、平板、手机等不同设备
- 移动端优化的卡片式布局

#### 4.4 设计风格
- **简洁专业**：界面设计简洁，专业感强，适合销售场景
- **无表情符号**：不使用 emoji 等装饰性图标，保持商务风格
- **清晰层次**：信息层次分明，重点突出
- **数据驱动**：以数据和图表为主，减少花哨装饰
- **易于阅读**：字体大小适中，对比度清晰，易于快速浏览

## 技术需求

### 1. 技术栈
- **前端框架**：React / Vue.js / Next.js 或纯静态HTML
- **样式**：简洁的CSS，专业商务风格
- **图表库**：用于性能对比（如需要）
- **部署**：GitHub Pages / Vercel / Netlify / AWS S3 + CloudFront


### 2. 数据来源
- AWS 官方文档和定价页面
- AWS EC2 API（可选，用于实时数据）
- 手动维护的 JSON 数据文件

### 4. 浏览器兼容性
- Chrome（最新版本及前两个版本）
- Firefox（最新版本及前两个版本）
- Safari（最新版本及前一个版本）
- Edge（最新版本）

## 非功能需求

### 3. 国际化
- 支持中英文切换
